PAGE 1 Ś Overview
Product Summary
This project is a backend automation system that manages subscription billing events automatically. It listens to Razorpay webhook events, processes them in n8n, updates subscription and payment records in Supabase, and sends customer emails like receipts, warnings, and cancellation messages.

Problem
Subscription businesses face common problems:

Payment success is not reflected in the app quickly
Failed payments are missed
Cancellations are not handled properly
Customers do not get proper billing emails
Billing data and app data go out of sync
Goal
Build a simple event-driven billing processor that:

Receives Razorpay webhooks
Updates subscription lifecycle in database
Sends billing-related emails
Handles failed payments and cancellations
Prevents duplicate webhook processing
Keeps database in sync with Razorpay
Users
Customers
Want receipts after payments
Want alerts if payment fails
Want confirmation when subscription changes
Admin / Product Owner
Wants accurate subscription records
Wants logs of all events
Wants visibility into failed payments and churn
In Scope
Razorpay webhook handling
Subscription activation, payment, failure, pause/halt, cancel
Supabase data updates
Email notifications
Retry handling for failed processing
Daily reconciliation with Razorpay API
Out of Scope
Frontend billing page
User authentication system
PayPal/Stripe integration
Full accounting/invoicing system
Refund creation flow

Event Types to Handle

subscription.activated
subscription.charged
subscription.pending
subscription.halted
subscription.cancelled
payment.authorized
payment.failed
Core Functional Requirements
1. Webhook Receiver
System must receive Razorpay webhook events
System must verify event signature in production
System must log every event in webhook_events
System must ignore duplicate events
2. Subscription Activation
When subscription.activated arrives:

Find or create user
Create subscription row
Link plan
Set status to active
Set user tier
Send welcome email
3. Successful Charge
When subscription.charged arrives:

Update subscription billing dates if available
Record payment in payment_history
Keep subscription active
Send receipt email
4. Payment Failure
When payment.failed or failed subscription charge happens:

Mark subscription as past_due or pending/halted based on event
Record failed payment
Send warning email
5. Subscription Halt / Pending
When subscription.pending or subscription.halted happens:

Update subscription status
Keep audit log
Notify customer if needed
6. Cancellation
When subscription.cancelled happens:

Mark subscription canceled
Set canceled date
Downgrade user to free tier
Send cancellation email
7. Retry and Recovery
Failed internal processing must be retried
System should not lose events if one step fails
8. Reconciliation
A daily workflow should compare active subscriptions in DB with Razorpay
If mismatch found, update DB and log it
Non-Functional Requirements
Webhook response should be fast
Data must be consistent
Processing should be idempotent
Emails should not block database updates
All timestamps stored in UTC

Success Criteria
Project is successful if:

Webhook events are received in n8n
Duplicate events are skipped
Subscription activation creates correct DB records
Successful payments are recorded
Failed payments update status correctly
Cancellation downgrades user
Emails are sent for major lifecycle events
Failed workflow items can be retried
Daily reconciliation can fix mismatches
Acceptance Tests
Trigger test subscription activation ? user + subscription created
Trigger successful charge ? payment row added
Trigger failed payment ? failed row + warning email
Trigger cancellation ? subscription canceled + user downgraded
Send same event twice ? processed once
Force mismatch ? reconciliation fixes it
Risks
Razorpay event payload may differ from expected shape
Some subscription data may require extra API call
Local webhook testing is less smooth than Stripe CLI
Wrong mapping of plan IDs can break upgrade/downgrade logic
Risk Handling
Inspect real webhook payloads in n8n execution logs
Store full event payload for debugging
Add retry workflow
Add daily reconciliation workflow
Final Result
This product gives a working automated billing backend for a subscription app using Razorpay, n8n, and Supabase.
